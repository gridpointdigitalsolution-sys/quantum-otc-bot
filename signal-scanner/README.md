# Quantum Signal Scanner — subscription signal feed

Sells **access to forex/gold trade signals** (entry · stop loss · take profit · reason) generated by
your PROVEN MT5 engines. Subscribers get the signals; they never get the EA. Your edge stays protected.

## How it works
```
Your VPS: MT5 + Signal EA on the proven pairs (live broker data, 24/5)
      │  setup fires
      ▼
QuantumForex_SIGNAL.ex5  ── POST entry/SL/TP/reason ──►  server.py (this hub)
                                                              │
                                              ┌───────────────┴───────────────┐
                                              ▼                               ▼
                                   Web dashboard (subscribers)     Telegram channel (subscribers)
```

## Files
- `server.py` — FastAPI hub: receives signals (`POST /signal`), stores them, broadcasts to Telegram, serves the dashboard.
- `dashboard.html` — branded live signal board (auto-refresh, mobile-clean).
- `../bots/Forex-SIGNAL/QuantumForex_SIGNAL.mq5` — the Signal EA (copy of the proven PHOENIX engine + signal emit layer; locked bots untouched).

## Deploy (your VPS — same pattern as the OTC bot)
1. Copy `signal-scanner/` to the VPS.
2. Create `secrets/` (gitignored) with:
   - `signal_secret.txt` — a random string (EA must send the same in `SignalSecret`).
   - `tg_token.txt` — your bot token. `tg_channel.txt` — your channel (`@yourchannel` or numeric id).
   - `pin.txt` — optional subscriber access PIN (until real billing).
3. `pip install fastapi uvicorn` → run `python server.py` (binds 127.0.0.1:8100).
4. Put nginx or a Cloudflare tunnel in front (same as the OTC dashboard) → public HTTPS URL.
5. In MT5: Tools → Options → Expert Advisors → Allow WebRequests → add your hub URL.
6. Attach `QuantumForex_SIGNAL.ex5` to each PROVEN pair's H1 chart, set:
   - `SendToHub=true`, `HubURL=https://<your-hub>/signal`, `SignalSecret=<same as the file>`.
   - Use each pair's proven settings. (Compile the .mq5 in MetaEditor first — F7 → 0 errors.)
